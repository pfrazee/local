Simple Ajax
Promises-based, with streaming, piping, and automatic body-parsing.

GET('https://myhost.com')
  .Accept('json')
  .then(function(res) {
  	res.status; // => 200
  	res.reason; // => 'OK'
    res.ContentType; // => 'application/json'
    res.body; // => { hello: 'world' }
  });

Virtual servers on the page
An emulation of HTTP lets you answer requests from client-side javascript.

local.at('#hi', function(req, res) {
  res.status(200, 'OK');
  res.ContentType('html');
  res.Link({ href: '#hi', rel: 'self service', title: 'Hello Service' });
  res.end('<h1>Hello, '+req.params.who+'</h1>');
});
GET('#hi?who=World').then(...);

Virtual servers in Web workers
Workers are loaded on-demand to answer requests to their hash-uris, then destroyed to preserve resources.

local.spawnWorker('/my-worker.js'); // optional
GET('/my-worker.js#hi?who=World').then(function(res) {
  	res.status; // => 200
  	res.reason; // => 'OK'
    res.ContentType; // => 'text/html'
    res.body; // => <h1>Hello, World</ht>
});

HATEOAS-driven
Entries from the Link header are searched and navigated by a programmatic user-agent.

var host = local.client('/');
GET(host.collection('users').item('bob')).then(function(res) {
  res.links.first('self');
  // => { rel: 'self item', href: 'https://myhost.com/users/bob', id: 'bob' }
});
